Introduction

GNU JAXP, part of the GNU Classpath Extensions project, is a free implementation of the standard XML processing APIs for Java:

  • SAX 2.0, the Streaming API for XML parsing,
  • DOM Level 2, the web standard API for processing XML, and
  • Sun's JAXP 1.1

Project status

GNU JAXP provides the Ælfred2 SAX2 parser, and is configured to use its optionally validating module by default. It includes the DOM Level 2 implementation provided by the World Wide Web Consortium. These are accessible through the JAXP 1.1 bootstrapping APIs.

At this writing, no XSLT support is bundled.

Documentation

You can browse the API documentation for GNU JAXP online here (no frames).

With JDK 1.2 or later the gnujaxp.jar file is most easily installed in $JAVA_HOME/jre/lib/ext; otherwise you may prefer to modify your CLASSPATH. If you have other JAXP parsers installed you may also wish to set up $JAVA_HOME/jre/lib/jaxp.properties so the two GNU JAXP drivers are used by default: 

# make JAXP clients prefer the GNU implementations
javax.xml.parsers.SAXParserFactory = gnu.xml.aelfred2.JAXPFactory
javax.xml.parsers.DocumentBuilderFactory = gnu.xml.dom.JAXPFactory

If you are using the portable SAX2 bootstrapping APIs, you may want to set the org.xml.sax.driver system property on each JVM invocation to be gnu.xml.aelfred2.SAXDriver (non-validating) or gnu.xml.aelfred2.XmlReader (optionally validating). Otherwise the org.xml.sax.helpers.XMLReaderFactory may not return the parser you expect.

More information will be provided later.

Help wanted!

Development of GNU JAXP is a volunteer effort, and you can also contribute! We need lots of help to finish the current APIs and to keep track of future enhancements. There are also opportunities for testers, web authors, and documentors. Please post to one of the classpathx mailing lists if you're interested helping.

Mailing lists

The main discussion list is <classpathx-xml@gnu.org>, which is used to discuss all aspects of GNU JAXP.

Announcements about GNU JAXP and other classpathx software are made on the low-volume <classpathx-announce@gnu.org> mailing list.

For subscription information, see the mailing list info from the ClasspathX project page.

Get the Software

You can download the latest software from ftp://ftp.gnu.org/pub/gnu/classpathx. You'll probably want to use one of the FTP mirror sites, although the mirrors may not be current.

At this writing, the current release is in a file called gnujaxp-1.0beta1.zip, which includes a pre-compiled JAR library, full javadoc, and source code. Make sure you use zip to unpack it, not jar, since some versions of "jar" don't handle execute permissions correctly. The download site is liable to change. Note that this is called 1.0beta1 mostly because it's the first release of this code from the GNU project. In fact, most of the code has been in use for several years now. We expect that the 1.0 release will include full support for building to native code using GCJ.

Interested parties can view up-to-the-second code in the jaxp module through CVS.

Report a bug

If you think you have found a bug in GNU JAXP, then you should send as complete a report as possible to <classpathx-xml@gnu.org>.
