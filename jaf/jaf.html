GNU-JAF: JavaTM Beans Activation Framework

The JAF is a system for declaring what beans operate on what MIME type data. Content handler beans can be defined to handle particular MIME content. The JAF unites internet standards for declaring content with JavaTM beans.


Downloading GNU-JAF

There are several ways of getting GNU-JAF:

GNU-JAF tar file

GNU-JAF jar file (just the class files)

Or you can download the CVS repository, see Classpathx project for more details.


Installing GNU-JAF

If your Java VM does not support extension library directories then you will need to reference the gnu-jaf.jar file in your CLASSPATH. If it does use extension library directories simply copy the jar file to the correct dir.


Documentation

The API documentation is available here.

The classes MailcapCommandMap and MimetypesFileTypeMap have a command line interface so that it's possible to tell what the GNU-JAF's understanding of mailcap and mime.types files are.


We need help

The main part of the JAF is finished but we don't have any beans that actually handle content in anyway. We'd like to know what sort of content handlers you would like... We'd like to build a large repository of beans that could be accessed over the internet. If you have an idea about a content type handler please contact Nic Ferrier (nferrier@gnu.org) and let him know.

If you're a programmer and you're interested in java beans why not help us out with building the content handlers? See the classpathx project page for more information.


Discussing the JAF

If you want to talk with other developers about JAF you can use classpathx discussion list. If you're not subscribed to this list you can do so by sending a mail with the subject subscribe to: classpathx-discuss-request@gnu.org.


Thank yous
