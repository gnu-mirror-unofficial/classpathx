Introduction to GNU Javamail

GNU Javamail, a part of the GNU Classpath Extensions project, is a free implementation of the JavaMail API specification, version 1.2. All of the code has been written from scratch without reference to Sun's code. This allows GNU Javamail to be used on a completely free operating system such as GNU/Linux. Although the code is targeted to work with free programs that implement the Java virtual machine, nothing prevents it from being used with any compliant VM.

GNU Javamail offers a protocol-agnostic framework for building mail client applications using the Java language. Protocol-specific libraries that are designed to work with GNU Javamail are called service providers. These enable GNU Javamail to work with standards such as POP3, IMAP4, SMTP and MIME. The providers currently under development appear below. If you don't see the one you want, sign up!

GNU Javamail is licensed under the terms of the GNU Lesser General Public License. Service providers may offer their contributions under that license or the GNU General Public License.

We need your help!

Development of GNU Javamail is a volunteer effort, and you can also contribute! We need lots of help to finish the current APIs and to keep track of future enhancements. There are also opportunities for testers, web authors, and documentors. Please contact nferrier@gnu.org if you're interested in contributing. (If you have ever looked at the source code for Sun's implementation of any of the APIs under development in GNU Javamail or any other classpathx project, you unfortunately cannot contribute code to that project. But your help in testing and documentation will be greatly appreciated.)

Project Status

  • Service Providers
    • POP3: 100% feature-complete.
    • IMAP4: Underway.
    • Mbox: 100% feature-complete.
    • SMTP: Underway.
    • NNTP: 100% feature-complete.

Downloading GNU Javamail

No public releases are yet available. Interested parties can view up-to-the-second code through CVS or other information on our page at savannah.

Mailing Lists

The main discussion list is <classpathx-javamail@gnu.org>, and is used to discuss all aspects of GNU Javamail, including development and porting.

Announcements about GNU Javamail and other classpathx software are made on the low-volume <classpathx-announce@gnu.org> mailing list.

To subscribe to any GNU Javamail mailing list, please send an empty mail with a Subject: header line of just "subscribe" to the relevant -request list. For example, to subscribe yourself to the main list, you would send mail to <classpathx-javamail-request@gnu.org> with no body and a Subject: header line of just "subscribe".

Report a Bug

If you think you have found a bug in GNU Javamail, then you should send as complete a report as possible to <classpathx-javamail@gnu.org>.
